## 四、 iOS / iPadOS WebKit 兼容开发注意事项（避坑红线）

在开发新交互面板时，请务必严守以下 WebKit 浏览器手势避坑守则，保证 iPad 平板及触屏的可按性与流程性：

1. **彻底移除 touchstart 绑定以防手势死锁**：
   - 现代移动端浏览器早已经在指定了 viewport 的页面中**完全取消了 300ms 点击延迟**。
   - 不要在按钮等交互元素上绑定 `touchstart` 并执行 `preventDefault()`，这会干扰 WebKit 底层的点击手势判定流，导致按钮在 iPad/iPhone 上完全“点不动、按了没反应”。
   - **开发规范**：所有新增按钮一律使用原生的 `<button>` 标签，且**只绑定 `click` 事件**，并采用 `e.stopPropagation()` 阻止事件向上传播至 3D 视角旋转的 touch 监听中即可。
2. **避让 iOS 独特的 Hover 模拟点击劫持**：
   - 触屏没有真正的 hover 指针。当 CSS 带有 `:hover` 伪类属性时，iOS Safari 会执行“第一次点击激活 hover，第二次点击才触发 click”的拦截机制。
   - **开发规范**：在移动端 CSS 样式中（如 `body.is-mobile` 的媒体查询下），必须显式将带有 hover 卡片面板彻底用 `display: none !important; pointer-events: none !important;` 禁用。
3. **滚动区域（.scrollable）防误触滚动**：
   - 项目全局为了防止手指滑动导致页面下拉刷新，对 `document` 的 `touchmove` 进行了 `preventDefault()` 强力拦截。
   - **开发规范**：你新增的面板内容若存在滚动，必须在包裹容器上指定 `.scrollable` 类，并且全局 `touchmove` 拦截器中必须排查 `e.target.closest('.scrollable')`，放行该容器的默认滚动。
